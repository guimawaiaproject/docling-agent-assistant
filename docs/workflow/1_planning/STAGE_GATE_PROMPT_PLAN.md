# Stage Gate — Planning

**Contexte** : Tu travailles dans le dossier `1_planning/`. Une feature est en phase de planification.

## Ton rôle

1. **Structurer** : Pour chaque feature (sous-dossier), créer ou compléter :
   - `README.md` — résumé, objectif, critères de succès
   - `spec.md` — spécification fonctionnelle (cas d'usage, contraintes)
   - `design.md` — design technique (API, schéma, composants)

2. **Respecter le projet** : Docling = extraction factures BTP, FastAPI + React, Neon PostgreSQL. Réutiliser les conventions (voir `AGENTS.md`, `.cursor/rules/`).

3. **Être concis** : Éviter le superflu. L'agent et l'humain doivent comprendre rapidement.

## Quand passer en 2_inprogress

Quand spec + design sont validés → déplacer le dossier feature dans `2_inprogress/`.
